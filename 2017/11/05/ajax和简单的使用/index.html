ajax和简单的使用

发表于 2017-11-05

AJAX的概述

* 什么是ajax:异步的数据请求,全称异步的javaScript and xml
* 了解同步和异步的区别:同步会刷新整个页面,异步刷新局部界面
* 功能:完成页面的局部刷新,不中断用户的体验
* xml:使用xml作为数据传递的格式,现在使用json

ajax的异步请求对象XMLHttpRequest

对象属性:
* readyState:xmlHttpRequest请求的状态
  状态码:0        未初始化--对象以建立,但是尚未初始化(尚未调用open方法)
        1        初始化----对象已建立,尚未调用send方法
        2        发送数据--send方法已经调用,但是当前的状态及http未知
        3        数据传送中-已接收部分数据,因为响应及http头不全,这时通过responseBody和responseTest获取部分数据会出现错误
        4        完成      数据接收完毕,此时可以通过responseBody和responseText获取完整的回应数据
* onreadystatechange:当XMLHttpRequest状态改变的时候触发一个函数
* status:获得响应的状态码.200,404..
* responseText:获得响应的文本数据
* responseXml:获得响应的xml的数据

方法

* open(请求方式,请求路径,是否异步):异步去向服务器发送请求
* send(请求参数):发送请求
* setRequestHeader(头信息,头的值):用于处理post请求方式

XMLHttpRequest对象的创建(了解)

* IE: 将XMLHttpRequest对象封装在一个ActiveXObject组件.
* Firefox:直接就可以创建XMLHttpRequest对象.
创建XMLHttpRequest对象的代码   
function createXmlHttp(){
   var xmlHttp;
   try{ // Firefox, Opera 8.0+, Safari
        xmlHttp=new XMLHttpRequest();
    }
    catch (e){
       try{// Internet Explorer
             xmlHttp=new ActiveXObject("Msxml2.XMLHTTP");
          }
        catch (e){
          try{
             xmlHttp=new ActiveXObject("Microsoft.XMLHTTP");
          }
          catch (e){}
          }
    }
 return xmlHttp;
 }

ajax的入门(了解)

编写步骤
1. 创建一个异步对象XMLHttpRequest对象
2. 设置监听事件,绑定回调函数:监听readyState(监听的状态码4)和status(监听状态码200)属性的状态
3. 设置向后台提交的路径open(请求的方式,请求路径,是否支持异步)
4. 发送请求send(参数)
get请求方式:
 function ajax_get(){
 //创建异步请求对象
 var xrt=createXmlHttp();
 //设置监听
 xrt.onreadystatechange=function(){
    if(xrt.readyState==4){//请求成功
        if(xrt.status==200){//响应成功
            document.getElementById("divtext").innerHTML=xrt.responseText;

        }
    }
 }
 //设置路径
 xrt.open("get","/day19ajax/AJAXServlet",true);
 xrt.send(null);
}

post请求方式:
function ajax_get() {
    var xrt = createXmlHttp();
    //设置监听
    xrt.onreadystatechange = function() {

        if (xrt.readyState == 4) {
            if (xrt.status == 200) {

                document.getElementById("divtext").innerHTML = xrt.responseText;
            }
        }
    }
    xrt.open("POST", "/day19ajax/AJAXServlet", true);
    xrt.setRequestHeader("Content-Type",
            "application/x-www-form-urlencoded");
    xrt.send("key=value")
}
post和get异步请求的区别:
get请求:请求参数拼接在地址栏后面:http://localhost:8080/xx/xxx?id=1
post请求:
需要另外设置setRequestHeader("Content-type","application/x-www-form-urlencoded")
(规范数据的提交格式:key=value,因为 request.getParameter(“key”))
请求参数在方法:send("id=1")

使用jquery的ajax

常用的异步请求方法:
$.get(url,param,function(data){},type):重点掌握
$.post(url,param,function(data){},type):重点掌握
$.ajax()和jquery对象.load()具体查询帮助文档
说明:
url: 浏览器访问服务器的请求路径
param: 浏览器发送请求时,提交的参数
function(data){}:使用回调函数的data,来接收服务器响应的数据
type: 服务器向浏览器响应数据的格式,数据格式:xml,json(这种格式,使用较多),还有 html, script, text, _default不常用。
注意:请求参数 param  {“name”:”jack”,”password”:”123”}
param参数的格式必须是json.

常见的json格式

json常见的数据格式有2种:
单个json对象:
var json={“name”:”jack”,”password”:”123”};
json数组
var jsonArr = [
                {“name”:”jack”,”password”:”123”},
                {“name”:”rose”,”password”:”666”}
]
得到rose  :   var  username =  json[1].name

解析json: json对象.属性名称.
var json={“name”:”jack”,”password”:”123”};
        var username = json.name

使用jquery的ajax异步请求验证用户名是否存在

步骤分析:
1. 导入jquery.js框架
2. 当用户填写完用户名时,发送异步请求到数据库,查看数据库中用户名是否存在
js代码实现:
     $(function() {
    //失去焦点时异步请求数据库查询用户名是否存在
    $("#textvalue").blur(function() {
        //获得用户输入的用户名
        var username = $(this).val();

        $.get("/day19ajax/RegisterServlet", {
            "username" : username
        }, function(data) {
            //服务器端设置当用户名存在时返回数组1,不存在返回数字0,并在浏览器进行判断
            if (data ==0) {
                //向页面提示用户名可以使用
                $("#spanId").html("<font color='green' >用户名可以使用</font>");
                //此时设置注册按钮不可用
                $("#but").attr("disabled",false);
            } else if
                (data == 1)
            {
                //向页面提示用户名已存在
                $("#spanId").html("<font color='red' >用户名已经存在</font>");
                //此时设置注册按钮不可用
                $("#but").attr("disabled",true);
            }

        });
    })
});
