视图

 视图是一种数据库对象,是从一个或者多个数据表或视图中导出的虚表,视图所对应
的数据并不是真正的存储在视图中,而是存储在所引用的数据表中,视图的结构和数据
是对数据表进行查询的结果

根据创建视图时给定的条件,视图可以是一个数据表的一部分,也可以是多个基表的联合
它存储了要执行检索的查询语句的定义,以便在引用该视图时使用

使用视图的优点:
1. 简化数据操作:视图可以简化用户处理数据的方式
2. 着重于特定数据:不必要的数据或敏感数据可以不出现在视图中
3. 视图提供了一个简单而有效的安全机制,可以定制不同用户对数据的访问
4. 提供向后兼容性:视图使用户能够在表的架构更改时为表创建向后兼容接口

视图的相关语法

创建:
create [or replace] [force] view 视图名
as select语句 
[with check option] 
[with read only]
说明:
or replace :若所创建的视图已经存在,Oracle自动重建该视图
force      :不管基表是否存在Oracle都会创建该视图
with check option 插入或修改的数据行必须满足视图定义的约束
with read only 该视图上不能进行任何dml操作


删除视图
drop view 视图名

简单视图

 --创建视图
create or replace view view_name
as select * from t_owners where ownertypeid=1;
 --利用视图进行查询
select * from view_name;//查询成功


补充视图其实是一个虚拟的表,它的数据来自于表中,更改视图中的数据,其实就是通过视图去更改表中的数据
,当视图中的数据发生改变时表中的数据也会发生变化
一个视图所存储的并不是数据,而是一条SQL语句

创建带有约束的视图

 --创建带有约束的视图
create or replace view view_name2 
as 
select * from t_owners where ownertypeid=1
with check option;
 --更新语句,查看约束效果
update view_name2 set name='张三' where id=1;//更新成功

 --再次更新,此时更新ownertypeid,查看约束效果
update view_name2 set ownertypeid=6 where name='张三';//受到约束更新失败

创建只读视图

 -- 创建只读视图,添加约束with read only
create or replace view view_name3 
as
select * from t_owners where ownertypeid=1
with read only;

 -- 这时进行修改操作
update view_name3 set name='李四' where id=1;--更新失败 
说明:只读视图只能进行查询操作

使用force关键字强制创建视图

创建一个视图如果SQL语句中的表不存在,但是以后可能会添加,这时可以加force
force 强制创建

create or replace force view view_name4
as
select * from random_table --random_table不存在,但此时可以创建成功

创建一个复杂视图

复杂视图是有聚合函数或多表关联查询的视图


create or replace view view_name4
as
select o.id 业主编号,o.name 业主名称,ot.name 业主类型
from t_owners o ,t_ownertype ot
where o.ownertypeid=ot.id;


--修改测试一
update view_name4 set 业主名称='范小兵' where 业主编号='1';--更新成功
--修改测试二,修改ownertypeid
update view_name4 set 业主类型='普通居民' where 业主编号='1';--更新失败
失败原因:修改的列'业主类型'不属于键保留表的列

键保留表

键保留表是理解连接视图修改限制的一个基本概念,该表的主键列全部显示在视图中,
并且它们的值在视图中都是唯一且非空,也就是说,**表的主键在一个连接视图中也是主键**
那么就称这个表是键保留表,**键保留表的字段可以更新,而非键保留表的字段不能更新**

在上面的例子中想要通过业主编号更新业主类型,但是业主类型所在表的主键并不在这个视图中
所以业主类型不能更新,修改的列'业主类型'不属于键保留表的列


基于这个原因,再次创建一个视图
create view view_accountsum as
select year,month,sum(money) moneysum
from t_accont
group by year,month
order by year,month;
此例中用到聚合函数,没有键保留表,所以不能更新
