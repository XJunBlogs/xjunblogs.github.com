EL表达式

发表于 2017-10-30

EL表达式:

1. 什么是EL:Expression language 表达式语言
2. 为什么学习EL:简化jsp的代码,而且减少<%%>
3. EL表达式的语法:${EL表达式}
4. EL的功能:
   获得数据(jsp的四个域)
   执行运算
   操作web开发的常用的对象
   调用java中方法:--很少用

使用el从jsp四个域对象中获得数据

el中有四个对象,分别对应jsp的四个域对象
pageScope         pageContext
requestScope      request
sessionScope      session
applicationScope  application
1. 获得单个数据(假设域对象已经存放数据了)
   ${pageScope.name}//获得pageContext域中键为name的值
   ${requestScope.name}//获得request域中name中的值
   ${sessionScope.name}//获得session域中name中的值
   ${applicationScope.name}//获得application域中name中的值
   简单写法:
   ${name}//它的作用是根据范围的从小到大的顺序获得域中的name的值,但只能获得一个域的值
2. 获得集合list与数组的值
  写法:${list[下标]} 和${arrays[下标]}
3. 获得map集合的值
  ${map.key}
  注意如果key有特殊字符如:.,-,+等等.写法应该为:${map["key"]}
  如果map的value为对象如user类里面有username属性写法应为:${map.key.username} 

  总结:
  1. el表达式写错了或没有找到,会返回一个空的字符串,而使用域对象.getAttribute      (name)此时会返回null
  2. el表达式获得对象的属性,这个对象必须有对象的getXXX()方法
  3. []用于有下标的数据(数组,list集合) .用于有属性的数据(map,对象)

使用el表达式进行运算

1. 执行算数运算
   <h3>EL执行算数运算</h3>
    <%
        pageContext.setAttribute("n1", "10");
        pageContext.setAttribute("n2", "20");
        pageContext.setAttribute("n3", "30");
        pageContext.setAttribute("n4", "40");
    %>
    ${n1+n2+n3}将值进行相加
2. 执行逻辑运算
<h3>EL执行逻辑运算</h3>
${ n1 < n2 } - ${ n1 lt n2 } <!-- less than --><br/>
${ n1 > n2 } - ${ n1 gt n2 } <!-- great than --><br/>
${ n1 <= n2 } - ${ n1 le n2 } <!-- less equal --><br/>
${ n1 >= n2 } - ${ n1 ge n2 } <!-- great equal --><br/>
${ n1 == n2 } - ${ n1 eq n2 } <!-- equal --><br/>
3. 执行关系运算
<h3>EL执行关系运算</h3>
${ n1<n2 && n3 < n4 } - ${ n1<n2 and n3 < n4 }<br/>
${ n1<n2 || n3 < n4 } - ${ n1<n2 or n3 < n4 }<br/>
${ !(n1 < n2) } - ${ not(n1<n2) }
4. 执行三元运算
<h3>EL执行三元运算</h3>

${ n1 < n2 ? "正确":"错误" }
5. empty运算    
${ user == null } - ${ empty user }
${ user != null } - ${ not empty user }

el操作web开发的对象11个

获得jsp中域的数据:pageScope,requestScope,sessionScope,applicationScope
接受参数:param,paramValues
获得请求头信息:header,headerValues
获取全局初始化参数:initParam
**web开发中的cookie:cookie,使用${cookie.名称.value}
**web开发中的pageContext:pageContext注意要会使用它获得项目路径${pageContext.request.contextPath}

jstl

1. 为什么学习jstl?
jstl和el结合替换页面中<%%>
2. JSTL版本:
    JSTL1.0            :不支持EL表达式.
    JSTL1.1 和 1.2        :支持EL表达式.
3. JSTL的标签库:包含了五类标签.
   core(核心标签),fmt(国际化标签),xml(XML标签),sql(SQL标签),fn(JSTL提供EL函数库)
4. 使用jstl:
   1. 引用jstl的相关的jar包(jstl.jar和standard.jar)
   2. 在页面中引入标签库<%@ taglib uri="" prefix=""%>

jstl的常用便签

1. <c:set var=”a” value=”123” scope=“request”></c:set>
   等同于： request.setAttribute(“a”,”123”);
2. <c:out value=”${a}” />
3. (重点)<c：if test=” el表达式”> 显示内容</c:if>
   如果el表达式返回结果为true，就显示标签之间的内容。
4. forEach遍历
    <c:forEach var=”a” items=”${域对象的名称}”>
            ${a}
    </c:forEach>
    = 遍历map，var指的就是map的entry（key-value）
    <c:forEach var=”a” items=”${域对象map名称}”>
            ${a.key}----${a.value}
    </c:forEach>
    上面都是增强for循环。
    普通for循环：
    <c:forEach var =”i” begin=”1” end=”100” step=”2” varStatus=”aa”>
            ${aa.index}--------${aa.count}
    </c:forEach>
